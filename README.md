# 🏋️ Workout Tracker Bot

Телеграм-бот на **aiogram 3.x** для ведення журналу тренувань.
Кидаєш усе тренування одним повідомленням — бот парсить, рахує тоннаж і оцінку 1ПМ,
зберігає в БД, будує графіки прогресу. Один код працює на SQLite (локально) і Postgres (хостинг).

## Формат запису

Одне повідомлення = одне тренування:

```
02.07.2026
жим лежачи: 60x10x3
присідання: 80x5x5
трицепс: 20x10x2 25x10x1
підтягування: 0x8x3
```

- Рядок вправи: `назва: вагаХповтореньХпідходів`
- Перший рядок з датою (`дд.мм.рррр`) — **необовʼязковий** (без нього = сьогодні)
- Кілька різних ваг на одну вправу — через пробіл: `трицепс: 20x10x2 25x10x1`
- Власна вага — пиши вагу `0`: `підтягування: 0x8x3`
- Роздільник чисел: `х` `x` `×` `*` — будь-який
- Назви автоматично зводяться до нижнього регістру (щоб збігались однакові)

## Команди й кнопки

- Нижня клавіатура: **Сьогодні**, **Тиждень**, **Вправи**, **Тоннаж**, **Видалити**, **Довідка**
- `/today`, `/week` — зведення за день / тиждень
- `/stats <вправа> [днів]` — макс. вага, оцінка 1ПМ, історія (можна за N днів)
- `/chart <вправа> [днів]` — 📈 графік прогресу; під ним кнопки періоду 7/30/90/Все
- `/volume [днів]` — 📊 тоннаж по днях + кнопки періоду
- `/compare A vs B` — ⚖️ дві вправи на одному графіку
- `/history` або `/delete` — список записів з кнопками ❌ для видалення конкретного
- `/undo` — видалити останній запис
- `/remind [днів|on|off]` — 🔔 нагадування, якщо давно не тренувався
- `/export` — вивантажити все у CSV

## Локальний запуск

```bash
pip install -r requirements.txt
export BOT_TOKEN=токен_від_BotFather      # Windows PowerShell: $env:BOT_TOKEN="..."
python bot.py
```

Без `DATABASE_URL` дані пишуться у файл `workouts.db` поряд.

## Деплой

> ⚠️ На безплатних хостингах диск часто стирається при рестарті — тому для проду
> під'єднай зовнішній Postgres (Neon/Supabase), і дані переживуть будь-який рестарт.

1. **БД:** заведи безплатний Postgres на neon.tech, скопіюй connection string у `DATABASE_URL`
   (префікс `postgres://` код сам перероблює під asyncpg).
2. **Хостинг:** бот працює через long polling, тож потрібен процес, що не засинає
   (Railway/Render *Background Worker*/Fly.io, власний VPS чи навіть Raspberry Pi).
3. У змінних оточення задай `BOT_TOKEN`, `DATABASE_URL`, `TZ=Europe/Paris`.
   Таблиці створяться самі при старті.

## Структура

| Файл | Призначення |
|------|-------------|
| `bot.py` | Хендлери, клавіатури, callback-кнопки, фонові нагадування |
| `parser.py` | Розбір повідомлення-тренування (дата + вправи) |
| `database.py` | Моделі та запити (async SQLAlchemy) |
| `charts.py` | Графіки прогресу / тоннажу / порівняння (matplotlib, Agg) |
| `requirements.txt` | Залежності |
| `Procfile` | Команда запуску для хостингу |
| `.env.example` | Приклад змінних оточення |
