# 🏋️ Workout Tracker Bot

Телеграм-бот на **aiogram 3.x** для ведення журналу тренувань.
Пишеш вправу текстом — бот її парсить, рахує тоннаж і оцінку 1ПМ, зберігає в БД.
Один код працює і на SQLite (локально), і на Postgres (на хостингу).

## Що вміє

- Записувати вправи в довільному форматі: `жим лежачи 60 3х10`
- Розуміє власну вагу (`підтягування 3х8`), дробову вагу, різні повторення в підходах (`жим 60 3х12,10,8`)
- `/today`, `/week` — зведення за день / тиждень з тоннажем
- `/stats <вправа>` — макс. вага, оцінка 1ПМ (формула Еплі), історія
- `/chart <вправа>` — 📈 графік прогресу ваги по датах (matplotlib), пунктиром оцінка 1ПМ; для вправ на власну вагу малює повторення
- `/volume [днів]` — 📊 стовпчиковий графік тоннажу по днях (типово 14)
- `/compare A vs B` — ⚖️ дві вправи на одному графіку
- `/remind [днів|on|off]` — 🔔 нагадування, якщо давно не тренувався
- `/list` — усі вправи, `/history` — останні записи
- `/undo` — видалити останній запис
- `/export` — вивантажити все у CSV

## Локальний запуск

```bash
pip install -r requirements.txt
export BOT_TOKEN=токен_від_BotFather      # Windows: set BOT_TOKEN=...
python bot.py
```

Без `DATABASE_URL` дані пишуться у файл `workouts.db` поряд.

## Деплой на безплатний хостинг

> ⚠️ Головне правило: **на безплатних хостингах диск часто стирається при кожному
> редеплої/рестарті.** Тому не тримай SQLite на диску хоста — під'єднай зовнішній
> безплатний Postgres, і дані переживуть будь-який рестарт.

### Крок 1. Безплатна БД (Neon або Supabase)

1. Зареєструйся на **neon.tech** (або supabase.com), створи проєкт.
2. Скопіюй connection string, схожий на:
   `postgresql://user:pass@ep-xxx.eu-central-1.aws.neon.tech/dbname`
3. Це значення піде у змінну `DATABASE_URL`.
   Драйвер `asyncpg` код підставляє сам — префікс міняти не треба.

### Крок 2. Хостинг бота

Бот працює через **long polling**, тому йому потрібен процес, що не «засинає».
Підійде будь-який варіант, де можна тримати `worker`:

- **Railway** (railway.app) — залий репозиторій, воно підхопить `Procfile`.
- **Fly.io** — `fly launch`, працює на free-allowance.
- **Render** — тип сервісу *Background Worker* (не Web, бо Web на free засинає).

Скрізь однаково:

1. Заливаєш цю папку в GitHub-репозиторій.
2. Підключаєш репо до хостингу.
3. У налаштуваннях (Variables / Environment) додаєш:
   - `BOT_TOKEN` = токен від @BotFather
   - `DATABASE_URL` = рядок із Neon/Supabase
   - `TZ` = `Europe/Paris`
4. Команда запуску: `python bot.py` (або хостинг сам візьме з `Procfile`).

Таблиці створюються автоматично при першому старті (`init_db`).

## Структура

| Файл | Призначення |
|------|-------------|
| `bot.py` | Хендлери команд і повідомлень |
| `parser.py` | Розбір рядків вправ у структуру |
| `database.py` | Моделі та запити (async SQLAlchemy) |
| `charts.py` | Побудова графіків прогресу (matplotlib, Agg) |
| `requirements.txt` | Залежності |
| `Procfile` | Команда запуску для хостингу |
| `.env.example` | Приклад змінних оточення |

## Формат запису

```
назва [вага] підходиХповторення
```

| Приклад | Розуміється як |
|---------|----------------|
| `присідання 80 5х5` | 80 кг, 5 підходів по 5 |
| `тяга 55кг 4x12` | 55 кг, 4×12 |
| `віджимання 3х20` | власна вага, 3×20 |
| `жим 60 3х10,8,6` | 60 кг, підходи 10/8/6 |

Роздільники підходів×повторень: `х` `x` `×` `*` — усі працюють.
